when to use
Long sequential tasks: AutoSync setup · creating a Task · connection setup · first-time org config. Don't use a wizard for <3 steps — a single form is fine. Don't use for non-linear flows — use sections + save.
horizontal
3–5 steps. Each step has a short label and a one-line sublabel. Done steps fill blue with a check; current uses a ringed dot; future is grey. Connectors fill blue as you progress.
vertical
≥6 steps, or steps with substantive content. Rail on the left with the current pane on the right. Active step has a Blue 50 background. Connectors are vertical lines between dots.
nav
Footer always shows Cancel (ghost, left) and Back / Next (right). Final step's primary turns into "Create" / "Start sync" — verb-led. Disable Next until current step validates. Don't auto-advance.
jumping
Allow clicking back to a completed step (badge shows check). Don't allow jumping forward over uncompleted steps. Don't reset downstream state when going back — preserve user input.
review step
Always end with a Review step listing all inputs as label/value pairs and a "Create" CTA. Each section has an inline "Edit" link that jumps back to that step.
save & exit
Long wizards (5+ steps) include a "Save draft" option in the header. Draft persists for 14 days. Reopening resumes at the last completed step.
a11y
Stepper is <nav aria-label="Steps"> with each step having aria-current="step" when current. Disabled forward steps are aria-disabled, not removed.
