affordance
Editable text shows a pencil icon on hover, plus a subtle grey background. Don't show the pencil at rest — it adds noise to every row. Hover state is enough to discover the affordance.
activate
Click anywhere on the value to enter edit mode. F2 or Enter from keyboard focus also enters edit mode. Field auto-selects the full value on entry so typing replaces it.
border
Active field uses the standard input style — 1px teal-600 border + focus ring. Inherits sibling typography (don't shrink to default 14px when editing a 22px heading).
commit
Save on Enter, blur, or check-button click. Cancel on Esc or X-button. Show keyboard hint inline while editing. Empty submit reverts to original value, never clears the field permanently.
optimistic
Update the cell visually on commit and show an inline "Saving…" chip. If save fails, revert the value and surface an inline error message in red below the cell — never a toast.
validation
Validate client-side on commit. Show error inline beneath the cell with row-tint red-50. Field stays in edit mode until valid or cancelled.
when not to
Inline edit only for label-like values (names, descriptions, tags, simple selects). For multi-field configuration, structured types, or anything requiring side-effects, open a drawer or modal instead.
a11y
Editable elements expose role="button" with aria-label="Edit {field}". Active input is a real <input>. Save/Cancel buttons have aria-label. Errors are role="alert".
