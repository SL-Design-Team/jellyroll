vs. select
Use a Combobox when there are many options (>~20) and users will type to find. Use a Select for short, fully-known option lists.
filtering
Local for <500 items (filter in JS). Async for remote/large datasets, with --validate-debounce (350ms) before firing. Show loading state via spinner replacing chevron + loading row in menu.
matching
Substring match by default, bolded in the option. Consider fuzzy match (Fuse.js or similar) for long names with abbreviations.
create-new
When async search returns empty, surface a "+ Create … " footer that resolves to a new entity. Critical for connection pickers.
keyboard
Type filters · ↑/↓ navigates · Enter selects active or creates · Esc closes without selecting · Tab selects active and closes.
a11y
role="combobox" aria-autocomplete="list" aria-expanded aria-activedescendant="option-id". Menu: role="listbox"; items: role="option".
