label
14/22 ExtraBold Indigo-900, sentence case. Always associated to control via for/id or wrapping.
required
Red-600 asterisk after label, aria-hidden="true" (the semantic comes from aria-required="true" on the control).
optional
Lowercase "(optional)" in subtle grey, 12px regular. Use when most fields in a form are required and a few are not. Don't use both required asterisks and optional labels in the same form.
control
40px default height · 12px horizontal padding · 4px radius · 1px Grey-400 border. Uses --field-* tokens — never invent new values.
prefix / suffix
Inline static text (URLs, units, currency). Non-interactive. Colored --color-text-subtle. Sits inside the same control container, separated from input by 8px gap.
icons
16px Lucide, inherits currentColor. Leading = context (search, link). Trailing = action (clear, dropdown, status).
helper text
12/18 below the control, subtle grey. Use for format hints and examples. Disappears (or moves to error) when validation fails.
error text
12/18 below the control, Red-700 with leading alert-circle. Always specific ("Must be a valid email") — never "Invalid input."
counter
11px monospace, right-aligned in the meta row. Turns Red-700 at limit. aria-live="polite" when approaching limit.
a11y
Label-to-control: <label for="id">. Error linkage: aria-invalid="true" aria-describedby="error-id". Required: aria-required="true".
