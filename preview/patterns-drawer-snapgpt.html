anchor
When SnapGPT is open, the Drawer's right edge anchors to the left edge of the SnapGPT panel, not the viewport. The Drawer sits inside the workspace column.
scrim
Scrim covers only the workspace area (between the left rail and the Drawer's left edge). SnapGPT is never scrimmed.
focus
Drawer traps focus within itself (X / form fields / footer buttons). SnapGPT remains a separate live region — clicking into the chat input shifts focus there without dismissing the Drawer. Esc closes the Drawer regardless of which region has focus.
state preservation
SnapGPT's chat thread, scroll position, draft prompt, and history persist across the Drawer open/close cycle. Never reset the panel when a Drawer opens.
responsive
At narrow viewport widths where Drawer + SnapGPT + min-workspace can't all fit (≤1280px), the Drawer takes precedence: SnapGPT auto-collapses to its toggle pill and restores when the Drawer closes.
do
Let users ask SnapGPT for help while a Drawer is open — that's the whole point of this pattern. Reference values from chat inside the Drawer form (copy/paste, drag-drop).
don't
Auto-close SnapGPT when a Drawer opens. Don't scrim SnapGPT — it implies the panel is inactive, which it isn't. Don't allow stacked Drawers on top of each other.
