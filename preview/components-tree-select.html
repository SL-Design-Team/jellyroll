variants
Multi-select (checkbox rows · tri-state parents · Select all header) — used for filtering by N pipelines / projects / snaplexes. Single-select (radio rows · click-to-pick · no Select all) — used for "move to folder", "pick parent", "choose snaplex".
row
Three slots in a fixed-width gutter: chevron (expand · hidden for leaves) · checkbox/radio · label (folder/branch icon + name). Trailing slot: optional count chip showing N / M selected or total.
indent
Each tree level adds 26px of left padding. Connector lines are not drawn — indentation + chevron carry hierarchy. Match the dom-shape: a child sits flush with its parent's label, not the chevron.
tri-state
Parent checkbox state is derived: checked if all descendants are selected · indeterminate if some are · empty if none. Clicking a parent's checkbox toggles all descendants together. Clicking the chevron only expands.
select all
Top "Select all" row (dashed bottom border) toggles every leaf in the tree. Inherits the same tri-state styling. Hidden in single-select mode.
search
Filters leaves and auto-expands ancestors that contain matches. Matching substrings bolded. Parent counts switch to "N matches". Footer offers "Select all matching" when results > 0.
footer
Left: Clear selection link (disabled when nothing selected). Right: live count "X / Y selected", or match summary when filtered. No Apply/Cancel — changes commit immediately like Multi-select.
field display
0 selected → placeholder. 1 selected → full path ("Sales analytics / Daily / orders-rollup"). 2–4 → individual chip tokens. 5+ → summary ("14 pipelines across 3 projects").
keyboard
↑/↓ traverse visible rows · → expand collapsed branch · ← collapse (or jump to parent if already collapsed) · Space toggle the row's checkbox/radio · Enter commit on single-select · Esc close. Typing focuses the search input.
perf
Virtualize after ~200 visible rows. Async branches load on first expand and cache; show inline spinner row beneath the parent while loading. Never load the full tree eagerly when it can have thousands of leaves.
a11y
Pop is role="tree" aria-multiselectable="true" (false for single). Each row: role="treeitem" aria-expanded aria-level aria-checked. Search input aria-controls the tree. Tri-state announces as "mixed".
