types
Action — triggers an action (filter, add) · Filter — multi-select toggle · Choice — single-select within a group · Input — token in a field, removable with X
states
Default · hover (border + text shift to Blue) · focus (teal halo) · selected (Blue-100 fill + Blue-600 border + ExtraBold) · disabled (muted grey, no-cursor)
sizes
Medium 32px · 13/16 · pad 6/10 (default)  |  Small 24px · 11/14 · pad 3/8
corner
4px (--radius-sm) — matches the SnapLogic system. Not pill-shaped.
icon
Optional leading icon (14px md, 12px sm). Filter chips use check when selected. Input chips end with a small x remove button.
a11y
Use <button> for action/filter/choice. aria-pressed="true" for filter, aria-selected="true" for choice (inside a listbox). Input chips: trailing X is its own button with aria-label="Remove …".
vs. Badge
Use a Chip when the user can interact with it (toggle, click, remove). Use a Badge for static metadata (status, tier, count) the user can't change.
