toolbar order
From left: search (always present) · "+ Add filter" · active filter buttons · view density / saved view to the right. Wrap to next row past 6 buttons.
search
Free-text search, scoped to the current view. Placeholder names the corpus and key facets ("1,240 pipelines by name, snap, or owner"). ⌘K focuses. Debounce 300ms (--validate-debounce token).
filter buttons
Resting buttons are subtle (grey border). Active filters show Blue 100 fill + Blue 600 border with a count pill of selected values. Click to open the popover; click a separate X-chip area to clear.
popover
Anchored under the button, 260px wide. Header title · in-popover search (when ≥10 options) · checkbox list with counts per facet · Clear / Apply at bottom. Apply commits; click-away cancels.
chips
One chip per facet (not per value) — "Status: Active, Failed" not two chips. Remove button on each. "Clear all" link on far right. Chip strip only appears when ≥1 filter is applied.
sort
Sortable column headers show chevrons-up-down at rest, switch to arrow-up/arrow-down when active. Only one column sorted at a time. Active sort header uses Indigo 900 text.
persist
Filters + sort persist in URL query string so deep links share. Reset only on explicit "Clear all" or app-level reset. Saved views are a separate pattern.
empty
If filters return zero rows, use the No results empty state with a "Clear filters" CTA — not the generic "No data" state.
