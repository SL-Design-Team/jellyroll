when to validate
On blur for required + format checks · on commit (Save / Next) for cross-field rules · on debounced change (--validate-debounce) for async checks like name-availability. Never validate while the user is typing — flashing errors mid-keystroke is hostile.
required marker
Red asterisk * after the label, with aria-required="true". Optional fields show a muted "(optional)" — never mark every required field and every optional one; pick one convention per form. SnapLogic convention: mark required.
error
Border --field-border-error, background --field-bg-error. Help text turns red ExtraBold with an alert-circle icon. Message is specific and actionable — never "Invalid input." Cite the offending characters when possible.
warning
Yellow border + yellow ExtraBold help text. For non-blocking concerns (cost implications, deprecations, unusual values). Doesn't prevent save.
async / checking
Teal border + spinning loader icon while the check runs ("Checking reachability…"). Resolves to success (green check, "Available") or error (red, specific failure). Debounce 300ms after last keystroke.
summary banner
On submit failure, render a red banner at the top of the form listing all errored fields as anchor links to the field. Headline counts errors ("3 fields need attention"). Focus moves to the banner, then the user can tab into links.
strength meter
Reserved for password / passphrase / API-key fields. 4 segments, colored Red → Yellow → Green. Label states the rule that would advance to the next level — never just "weak / strong" without guidance.
positioning
Help / error text always beneath the field, never beside it (alignment breaks at narrow widths). Character counters go bottom-right of the field, opposite the help.
save guards
Save button stays enabled until clicked — disabled-on-error pattern hides why the form can't submit. On click, validate everything, show errors, focus the first.
a11y
Each field uses aria-describedby pointing to its help/error node. Errored field gets aria-invalid="true". Banner is role="alert". Async-checking states use aria-live="polite".
