Status in data vizwhen a chart encodes status, pull from semantic tokens — not the vibrant categorical
Semantic mapping
Succeeded--sl-green-600
Failed--sl-red-600
Warning--sl-yellow-600
Running--sl-blue-600
Queued / Idle--sl-grey-500
These are the same tokens used by the Status tag component. A chart of "pipeline runs by status" pulls from this set, not the vibrant categorical palette.
Decision rules
✓ Chart encodes status → use semantic tokens
✓ Chart shows unordered categories (regions, teams, snap types) → use vibrant categorical
✓ Chart shows magnitude / one variable → use vibrant sequential
✓ Chart shows ± a baseline → use diverging
✗ Don't mix palettes in one chart (a single red series in a categorical-palette chart reads as "broken")
✗ Don't reuse semantic tokens for non-status data (red doesn't mean "category 7")
