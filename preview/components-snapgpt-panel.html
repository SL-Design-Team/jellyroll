DESIGNER
workspace
SnapGPT
width
400px default · resizable 320–600px by dragging the left edge. State persists per user.
behavior
Pushes workspace content left — does not overlay. No scrim. Persists across navigation; the chat thread stays alive. Toggled from the global header pill or via keyboard shortcut.
chrome
Header: 56px (matches the global application header) · Blue-900 dark band with "SnapGPT" wordmark (gradient on "GPT"), new-chat / suggestions / history / more / collapse icons. Footer: prompt input with attach, suggestions, project selector, send button.
edge
1px gradient edge (Teal-400 → Blue-600) on the panel's left side — the SnapGPT brand accent, separating panel from workspace.
motion
Slide-in 280ms cubic-bezier(0.2, 0, 0, 1). Workspace reflows at the same rate (uses CSS grid, not absolute positioning).
a11y
Not a modal — does not trap focus. aria-label="SnapGPT chat panel" on the region. Toggle button has aria-expanded and aria-controls.
with Drawer
A config Drawer opens inside the workspace area, anchored to this panel's left edge. The scrim covers only the workspace — SnapGPT remains uncovered and fully interactive, so users can keep chatting while configuring. See Drawer + SnapGPT pattern card.
