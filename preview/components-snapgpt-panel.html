Here's more about the recommended workflow:

Start with Preview Data Insights so SnapGPT can understand your actual data before helping you build.

  1. Validate your pipeline to generate preview data
  2. Right-click a Snap in the pipeline canvas
  3. Select Get Preview Data Insights in SnapGPT
  4. Use Plan mode to review what SnapGPT understands and define what you want to build
  5. Use Generate to create the pipeline or workflow with better context

Preview Data Insights can help identify data shape, quality issues, important fields, PII/compliance signals, and business logic patterns.

workspace
SnapGPT
width
400px default · resizable 320–600px by dragging the left edge. State persists per user.
behavior
Pushes workspace content left — does not overlay. No scrim. Persists across navigation; the chat thread stays alive. Toggled from the global header pill or keyboard shortcut.
header
56px · Blue-1000 · "SnapGPT" wordmark · teal→blue hairline gradient on the bottom edge. Icons: new chat (square-pen), more (⋮), close (✕).
messages
User bubbles: Grey-100 bg · AI bubbles: white with Grey-200 border. Both show a 24px avatar icon + timestamp above the bubble. AI responses support bold, ordered lists, and suggested follow-up chips beneath a "What's next?" heading.
input
Bordered input box (Blue-400 ring) · "Ask SnapGPT…" placeholder · toolbar row: + (attach), Mode dropdown, Select context dropdown, teal circular send button.
edge
2px gradient edge (Teal-400 → Blue-600) on the panel's left side — the SnapGPT brand accent, separating panel from workspace.
motion
Slide-in 280ms cubic-bezier(0.2, 0, 0, 1). Workspace reflows at the same rate via CSS grid.
a11y
Not a modal — does not trap focus. aria-label="SnapGPT chat panel" on the region. Toggle button has aria-expanded and aria-controls.
with Drawer
A config Drawer opens inside the workspace area. The scrim covers only the workspace — SnapGPT stays uncovered so users can keep chatting while configuring. See Drawer + SnapGPT pattern card.
