severity
Pick the smallest container that fully scopes the failure. Inline < Section < Banner < Branded page. Don't wrap a page in an error state if only one panel failed.
branded page
Reserved for routing-level errors: 404 / 403 / 401 / 400 / 500 / 503. The signature mark is the "Oops!" headline with two interlocking teal rings as the leading O — the snap mark. Always paired with the dark Indigo 900 navbar showing the error code in the title.
action area
By error code: 404 / 403 show product launchers (Monitor / Designer / AutoSync) so the user can recover sideways. 401 shows "Return to login page". 400 shows "Return to previous page". 500 shows a contact-support link.
copy
Headline is a complete sentence ending in a period, sentence case, leads with "Sorry" only on 4xx (the user's intent failed). 500 uses "Well, this is unexpected…" — calmer self-deprecation. Always paired with one descriptive sub-line.
color
Red is for errors — something failed or is blocked. Yellow is for warnings — something needs attention but isn't broken yet (expiring credentials, deprecation). Don't use red for warnings.
recovery
Retry buttons should attempt the failed action without reloading the page. If the error is permanent (404, 403), the action goes back to a known-good route — never just "Retry" into the same failure.
a11y
Container uses role="alert" (assertive) for new errors, role="status" for ongoing/repaired states. Focus moves to the error region when it appears. Icons are aria-hidden; the text carries the meaning.
