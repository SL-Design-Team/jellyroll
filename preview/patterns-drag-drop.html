handle
Always provide an explicit grip icon (grip-vertical) when rows or columns are draggable. Don't make the whole row draggable — conflicts with click-to-select. Cursor is grab at rest, grabbing while dragging.
dragging row
The original row gets a subtle elevation: Blue 600 border, raised shadow, small rotation (~−0.5°) and 2px offset. The row remains in place as the ghost — don't remove it; the insertion-line tells the user where the drop will land.
insertion line
2px Blue 600 line with a 8px filled dot on the left, rendered between target rows. Snaps to row gaps, never overlaps content. Only one line visible at a time.
canvas drop
Designer-style: dashed Blue 600 outline + Blue 50 fill on valid drop targets. Show "Drop here" with a + icon, sized to match the dropped node. Invalid zones (full lanes, type mismatch) don't highlight at all — never use red on hover.
floating ghost
When dragging from palette to canvas, render a floating ghost that tracks the pointer: white card with Blue 600 border, slight rotation, raised shadow. Hidden when over an invalid zone.
file drop zone
Dashed Blue 400 border + Blue 50 fill at rest. Active (file over zone): Blue 600 solid border + Blue 100 fill + name the file. Rejected: red border + red fill + specific reason ("This zone only accepts JSON, YAML, or AVRO").
keyboard
Every draggable affordance must have a keyboard equivalent: rows reorder via focus + Space to pick up, ↑/↓ to move, Space to drop, Esc to cancel. Files have a "Browse" link. Canvas placement has the snap palette's "+" button.
announce
Use aria-live="assertive" to announce pickup ("Mapper picked up"), movement ("Mapper, position 2 of 4"), and drop ("Mapper dropped, position 3 of 4"). Cancel announces "Drop cancelled".
auto-scroll
When dragging near a scrollable edge, scroll the container at 8px/frame max — never the page. Stop scrolling when pointer moves away from edge.
