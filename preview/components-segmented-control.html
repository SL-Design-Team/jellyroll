shape
Sunken track with a 3px inset pad and 1px outer border. Outer corners use 4px radius; the selected pill nested inside also uses 4px radius. Selected = Blue 600 pill; unselected = transparent.
surfaces
The track stays transparent on either surface; only the border, idle text, and hover fill swap. Add sc--inverse on dark surfaces (Designer rail, SnapGPT panel, Indigo 1000 nav). The selected Blue 600 pill is identical in both modes.
states
Idle: Grey 700 (light) / Grey 400 (dark). Hover: text darkens / brightens with a subtle fill (Blue 6% on light · white 6% on dark). Selected: Blue 600 fill, white text. Disabled: Grey 400, no hover. Focus: 2px teal outline on the segment only.
sizes
md (default) · 30px tall, 13px ExtraBold, 14px icon, 14px segment padding. sm · 24px tall, 12px ExtraBold, 12px icon, 10px padding. Toolbar = sm; page-level filter = md.
content
Icon + label, label only, or icon only. Always show labels when space allows. Icon-only is reserved for dense rails — give every segment an aria-label and a tooltip.
when
2–5 mutually-exclusive options the user toggles frequently and wants visible at all times (view modes, time ranges, density, filter modes). Not for one-off pickers (use Select) or multi-select (use Multi-select).
vs. tabs
Use Tabs to switch content panels at the page or section level. Use Segmented control to filter/transform an existing view. Tabs change what; segments change how.
keyboard
Tab enters the group, focus lands on the selected item. ←/→ moves selection (commits immediately, single tab-stop per WAI-ARIA radio pattern). Home/End jumps to first/last.
a11y
Outer container role="radiogroup" with an aria-label. Each item role="radio" aria-checked. Disabled segments use aria-disabled="true" so they remain focusable.
