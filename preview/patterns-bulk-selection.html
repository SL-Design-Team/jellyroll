checkbox col
36px-wide first column. Header checkbox state: empty · indeterminate (mid-rule bar) · checked. Clicking header toggles all rows currently in view, never the full filter set.
action bar
Appears above the table when ≥1 row is selected (don't replace the header — both stay visible). Bar uses SL Blue 100 background with Blue 300 bottom border. Pinned, scrolls with table head.
selected row
Background tint SL Blue 50; checkbox stays checked. No left stripe — the background tint, checkbox state, and bulk-action bar above are sufficient signal. Don't bold the row text — color shift is enough.
actions
Most common bulk actions first (enable/pause/tag). Destructive actions (delete, archive) live last with red border + icon. Disabled actions show a tooltip explaining why ("Cannot delete — 3 selected are running").
select all
Header checkbox selects only visible rows. After that, show a banner offering to "Select all N matching your filters". Two-step prevents accidental org-wide actions.
clear
Always a "Clear selection" text link on the bar's right side. Esc clears. Switching tabs / filters preserves selection if rows still match; otherwise clears.
shift-click
Shift-click selects a range from the last-clicked checkbox. ⌘A selects all in view.
a11y
Checkboxes are real <input type="checkbox"> with aria-label. Header uses aria-checked="mixed" when indeterminate. Bar count is announced via aria-live="polite".
