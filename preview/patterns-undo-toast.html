when to use
Any reversible action: soft-delete, pause, archive, move, bulk tag, dismiss notification, mark resolved. Use instead of a confirmation modal when the action is cheap to undo — confirmations interrupt; undo respects user flow.
built on
Uses the standard Toast component. Variant: info for the undo window, loading while reverting, success on restore complete. Never red — for errors, use the inline Error states pattern.
placement
Bottom-center of the viewport, 18px gutter (matches Toast). Stacks vertically when multiple toasts. Don't pin to bottom-right (Designer canvas chrome lives there) and never block the form/footer action bar.
copy
Past-tense sentence stating what happened, with the object name in quotes and bold. Single sentence — no meta line. Scope is named inline when relevant ("in Sales analytics"). Never apologetic, never "Are you sure?" — the action is already done.
undo button
Outline pill, scoped to the toast's accent (Blue 600 on info). Label: "Undo" for single, "Undo all" for bulk. Keyboard: ⌘Z triggers the most recent toast's undo while it's visible.
countdown
2px accent-colored bar pinned to the bottom edge of the toast, shrinking left-to-right over the window. Default 10s for undo toasts (longer than the Toast component's 3s default — undo needs time to find). Hover pauses; mouse-leave resumes.
bulk
One toast for the whole batch — never N toasts. Show count + scope inline ("47 pipelines in Sales analytics"). "Undo all" restores everything atomically; if any fail to restore, show an error toast listing failed items.
after window
Once the window expires, the toast either auto-dismisses or shifts into a "Restored from Recycle Bin" affordance reachable via the bin — never silently lose access to recovery.
not for
Non-reversible actions (permanent deletes, environment teardown, password reset). For those, use the Destructive confirm dialog with type-to-confirm.
a11y
role="status" with aria-live="polite"; toast appearance is announced. Undo button is a real <button>, reachable by Tab while toast is visible. Focus is not stolen — toast appears non-modally.
