Counts are numeric ("12", "99+"). Badges are short labels ("Beta", "New", "Pro") — UPPERCASE 10px ExtraBold matching the system's pill style. Chip color adapts to the parent button.
Dropdown buttons use chevron-down as the right icon — visually identical to icon-right, semantically distinct (opens a menu vs. triggers an action).
Split buttons share a height and corners. Divider is a 1.5px white-translucent inset on primary, the existing border on secondary, a 1px Grey-300 line on tertiary. The trigger half always has an aria-label plus aria-haspopup="menu" — it opens the Menu component (light, on-canvas), not the dark header dropdown.
Large (40px) · page actions, modal footers, primary CTAs  |  Medium (36px) · toolbars, inline forms, dense panels  |  Small (28px) · table row actions, chips, condensed property editors
Icon-left for back / status / object-type prefix. Icon-right for forward / open / disclosure. Never both at once. Icons inherit currentColor.
Icon-only buttons are square (40 / 36 / 28). Always include aria-label describing the action.
Disabled uses tinted blue (Blue-200 fill / Blue-300 text) — keeps the blue identity rather than greying out. Cursor is not-allowed.
Loading swaps the leading icon for a sl-spinner (sm, inherits currentColor) and keeps the button full-color — not the disabled tint. Set aria-busy="true", use the progress cursor, and keep the resting width so the layout doesn't jump. Primary stays solid blue while loading (the gradient is reserved for hover).
