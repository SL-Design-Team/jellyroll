structure
Inline-flex container of tooltip-wrapped Avatar primitives. DOM and visual order both run left-to-right; place the overflow button last so it forms the right end-cap.
ring
2px solid in the surface color (white on canvas, Grey-100 on sunken, Blue-1000 on navbar). Acts as a "cutout" so adjacent avatars read as distinct shapes.
overlap
-4 / -6 / -8px for xs / sm / md.
hover / focus
After 300ms, show the visible user's full name in a Tooltip above the avatar. Raise only the hovered item above its neighbors; do not change its fill, outline, scale, or shadow. Each avatar is keyboard-focusable (tabindex="0"): focus shows the same name Tooltip and adds the standard teal ring, so keyboard users reach every name.
presence
A status dot sits at the avatar's bottom-right — filled Green-600 for active, Grey-500 for inactive — with a 2px surface-colored ring (and a raised z-index so it clears the next, overlapping avatar). Active is the default; add sl-avatar--inactive to also neutralize the avatar on light surfaces or dim its identity color in the global header. The dot is never the only signal: include Inactive in the tooltip and accessible name, and in the member list.
color
Use soft 100-step fills with 700/800 text in tables and other light contexts. On the dark global header, use saturated 600/700-step fills with white initials.
overflow
Show ~5 avatars then a native "+N" button — same shape, Grey-200 background, indigo text. Hover shifts the fill to Grey-300 and shows "Show N more members"; focus uses the standard teal ring. Clicking opens a member-list Popover: white card, 1px Grey-300 border, radius-sm, raised shadow, min-width 240px, no header. Each row is an xs avatar + full name, with a muted right-aligned presence label ("Inactive") where applicable; the list scrolls after ~5 rows (max-height 200px). Escape closes it and returns focus to the button.
a11y
Each visible Avatar carries aria-label="Full name", including , inactive when applicable, and is focusable so keyboard users see the name Tooltip. The overflow control is a native button with aria-label="Show N more members". When wired to the Popover, also set aria-haspopup="dialog", aria-expanded, and aria-controls; Enter and Space activate it without custom keyboard handling.
