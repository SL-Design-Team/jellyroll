structure
Inline-flex container of tooltip-wrapped Avatar primitives. DOM and visual order both run left-to-right; place the overflow button last so it forms the right end-cap.
ring
2px solid in the surface color (white on canvas, Grey-100 on sunken, Blue-1000 on navbar). Acts as a "cutout" so adjacent avatars read as distinct shapes.
overlap
-4 / -6 / -8px for xs / sm / md.
hover
After 300ms, show the visible user's full name in a Tooltip above the avatar. Raise only the hovered item above its neighbors; do not change its fill, outline, scale, or shadow.
presence
Active is the default. Add sl-avatar--inactive to neutralize the avatar on light surfaces or dim and desaturate its identity color in the global header. Include Inactive in its tooltip and accessible name.
color
Use soft 100-step fills with 700/800 text in tables and other light contexts. On the dark global header, use saturated 600/700-step fills with white initials.
overflow
Show ~5 avatars then a native "+N" button — same shape, Grey-200 background, indigo text. Hover shifts the fill to Grey-300 and shows "Show N more members"; focus uses the standard teal ring. Clicking opens a member-list Popover with each person's name and presence status.
a11y
Each visible Avatar carries aria-label="Full name", including , inactive when applicable. The overflow control is a native button with aria-label="Show N more members". When wired to the Popover, also set aria-haspopup="dialog", aria-expanded, and aria-controls; Enter and Space activate it without custom keyboard handling.
