when
Graphs where a regular force-directed Network renders as spaghetti — typically >100 nodes with cross-cluster edges. Best when nodes naturally divide into 2 or 3 categories (Source / Pipeline / Destination · Provider / Consumer · Read / Compute / Write).
axes
2 or 3 categorical axes radiating from a center, equally spaced (180° for 2, 120° for 3). Each axis is a category; node position along the axis is a continuous metric (degree, volume, age, etc.). Both ends of the axis must be labelled — show the metric direction.
node
Filled circle, color = axis category. Size scales with a secondary metric (volume, criticality). Outermost positions are highest-value. Don't label every node — show only top hubs + on hover/selection.
edge
Quadratic bezier curve with control point near the center, creating the trademark "petal" arcs. Stroke color = source axis color. Width / opacity scale with edge weight. Always non-directed unless arrowheads are needed.
direction
When edges have direction, arrowhead at the destination end (small filled triangle, same color as edge). For our pipeline flow plots, edges flow Source → Pipeline → Destination by convention.
same-axis
Edges within the same axis are not allowed in a hive plot. If you have intra-category edges, either omit or use a duplicate axis (e.g. Sources A and Sources B) at slightly offset angles.
position scale
Log-scale by default for skewed distributions (rows/day, requests). Linear when range is <1 decade. Always note the scale in a footer caption ("axis position: log(rows / day)").
selection
Click a node to focus: dim all non-neighbour nodes + edges to 0.2 opacity, keep selected node + 1-hop edges + 1-hop nodes at full opacity. Dashed halo around the selected node. Click empty canvas to clear.
node ribbons
When position alone doesn't convey the secondary metric clearly, layer concentric tier rings behind the axes — each ring = one bucket (log decade, status, etc.) and tints sequentially. Rings stay in the green ramp so cluster colors on nodes stay distinguishable. Don't use rings AND tick labels; pick one calibration system per chart.
vs. network
Use Network graph when structure is exploratory and clusters emerge from layout. Use Hive plot when categories are known up-front and you need a stable, comparable view across datasets. Pair them: Network for discovery, Hive for reports.
a11y
Always pair with a sortable table view of nodes + their connections (toggle in toolbar). Each axis labelled with aria-label; nodes are role="button" aria-label="{name}, axis {category}, position {value}". Colorblind-safe palette (Orange / Blue / Teal). Pattern overlays on stroke when used as the only differentiator in print export.
